MY ROLE

I was hired as a Junior/Full Game Developer. My main role was to develop systems for the Pipeline that would be used in other future games from the Studio. Some of the main ones were the Tutorial System, PopUp System, UI Canvas System, among others.

I joined mainly for the opportunity to learn and work in a team. I already had solid knowledge in several areas, but because I had worked in several positions without a team, there was no one to talk to about the systems and find out if it was the best option. After this experience and having learned a lot from the Lead Game Developer and other colleagues, I reached a point where I am very confident in my knowledge, in addition to having learned a lot about working in a team of developers.

As I stayed and delivered my tasks, I also received more trust from the other developers. I started checking the quality of the game by making bug reports, as well as solving some bugs and helping the other devs to also fix other bugs with per-programming, trust in important screens and systems, and reviewing PRs.

I also set up A/B tests to analyze data later via Firebase analytics.
